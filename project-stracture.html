Setup Guide

  • Download the whole project by using your favorite GIT CLI or GUI (for CLI the command is: git clone https://github.com/teddy4445/lecture_website_template.git
  • Open the folder with the project (if you are using Windows and CLI enter "cd lecture_website_template"
  • Inside the project, create new folder named "lecture_website_template" (yes, just as the name of the repo). If you are using Windows and CLI just enter "mkdir lecture_website_template"
  • Move all the project's folders (js, img, fonts, data, css) into the "lecture_website_template" folder you just created. (no ready CLI commend unfortunately).
  • At this point you are ready to run the project. You can use any server you wish. We recommended the "Web Server for Chrome" which can be downloaded here
  • After opening the webserver, configure it to the root folder of this project. If you are using the "Web Server for Chrome" server just open it and select the root folder as shown here
  • Finally, open the server in the root (Usully http://127.0.0.1:8887) to get into the index page of the project (for example by clicking on the link as shown here)

Code Design Guide

  • All files and folders are Cabab style (word-word-word)
  • The HTML is HTML5 - try to use tags from HTML5 and not 4 or 3 (you can check in wwwschool)
  • The CSS is CSS3 - try to use the new features of V.3 like animentions and others and not JS.
  • For the grid we use FlexBox - you can learn more about it in: https://css-tricks.com/snippets/css/a-guide-to-flexbox/
  • All JS files (except main.js) must be classes (OOP)
  • Each importent section in the HTML must start with a one line comment which describes the section
  • Each Js function must has at least one line documentation (if not trivial) and full description if it is a complex function
  • All JS components inherit from the 'Element' class and importent it.

Stracture

  • In order that the website will presented currectly in GitHub pages put all the folders inside a folder with the same name as the Repo. For example: "lecture_website_template"
  • all HTML files in the root folder
  • Folder "components" contains all static HTML components we just need to inject on page load which have been generated ahead by the "generator" service.
  • Folder "css" - all design (css) files of the project. Try avoiding spliting to multiple files and use the 'main.css'
  • Folder "data" - all data files (json & txt) which the website render on page load and generated by the "generator" service. Each page uses one or more data file.
  • Folder "fiels" - all the user's files which are not images (any format) uploaded by the user in the "generator" service and used in the website.
  • Folder "fonts" - all the fonts files used in the website.
  • Folder "img" - all the imges files used in the website.
  • Folder "js" - all the JS files in the website. 'main.js' is used in all pages and load the components (components folder) and other general logic. Inside, js/components folder has classes to read and generate each component used in the pages. js/Errors folder use project spesific JS errors for easy debuging. js/pages folder has a Js file per page used to generate this page.
