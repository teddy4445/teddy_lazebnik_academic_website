The Full Story

Join us in the journy of medical nanorobotics in the blood system. In this journy we will review the research steps needed to develop multirobotics medical treatmenat protocol.

We will review the following steps:

Motivation

Imagine yourself that a paitent comes to the doctor's office (or doing an online meeting). The doctor (or AI platform) assess the paitent's condition and decides on the sutable treatmenat protocol.

In the case where the paitent needs drug-based treatmenat the doctor will enter into the computer both the paitent's personal data and the requested medicine. The computer will generate a recipe for a printer to print a personalized medical nanorobotics (MNRs) drug.

By a simple injection of the capsule's content to the bloodstream of the paitent, the doctor can proceed to the next paitent, knowing that the MNRs taking care on this paitent.

Our Scope

There is active research on both AI platform to replace doctors and biological printers. These subjects are out of the scope of our work.

We are intrested in the part from the moment the user (either human or AI doctor) is describing the needed treatmenat protocol and providing the paitent's personal data and up to the moment where the computer produce the recipe for the MNRs capsule.

It is possible to divide this process into 4 main components. First, a programming languge for the user. Second, a library of MNR that can be used in the recipe. Third, a compiler generating a MNR recipe from the user's progam and the library. Fourth, a simulator to make sure the recipe is correct and potensioly improve it.

The Tolkien Project

The Tolkien Project is the name of this research and includes all the components described.

At the beginning, Inbal Wiesel-Kapah develop the Athelas programming language and the Bilbo compiler. Wiesel-Kapan et al. develop a rule-based programming language allowing the user to abstractly design the action of the population of different MNRs. The Bilbo compiler was able to provide a recipe of multiple types of MNRs with different payloads.

It is possible to add to Wiesel-Kapan et al.'s model several elements to advance the model for what we want to achieve.

Athelas Language

The Athelas Language developed by Wiesel-Kapan et al. is a rule-based programming paradigm. Each statment in Athelas is made of 4 clauses: Initialize, When, Actions, Until.

Athelas allowing to design which MNRs will operate (aka. expose a payload) according to some condition on abstract location in the body and fulfillment of some condition.

Bilbo Compiler

The Bilbo compiler developed by Wiesel-Kapan et al. getting as input a Athelas-written program and returns a list of MNRs with the right parameters (spesificly, their payload).
The algorithm behind Bilbo shown to be complete, sound, and halts.

Bilbo is not able to tell the right amount of MNRs per MNR type with the spesific parameters. In addition, same MNR can be different in multiple parameters except there payload.

Moreover, Bilbo is not able to make sure the actions, conditions or location are biologicaly valid per the assigned MNR.

Frodo Compiler

The Frodo compiler is the final goal of the Tolkin project. Frodo compiler should extend the Bilbo compiler to handle automatic usage of library of MNRs with big range or chemical, biological and physical properties.

In addition, the Frodo compiler will take advantage on in-silico (simulated) performance of the suggested result to optimize it.
The outcome of Frodo is almost identical to the one from Bilbo; Frodo adds the amount of MNRs needed to perform the requested task.

Currently, Frodo Compiler is not yet developed as it depands on several other components first, which will be reviewed now...

Elrond's Library

MNRs are very different from each other. By reviewing 200 academic papers, using domain experts, we manully extract the following list of major properties:
core meterial, size, magnetic, shape, charge, membrance, cell-penetration, surface area, toxicity, payload, treatmenat or \ and analysis, used for in-vivo or \ and in-vitro.

Elrond's library is a system that takes academic papers in English in PDF file format and extracting instances of MNRs with there properties and values. After extracting the information, the system organize it a rational database, while marging properties of the same MNR types and alerting confilicts in values for the same property.

Currently, a DB with 100 records and 5 properties has been extracted with a good accuracy. Multiple open source projects have been published from this research.

Finding the articles needed for the Elrond's library was a challging task by itself. Existing search engines was not good enough for the task.

Rivendell

Rivendell is an academic meta-search engine working on Pubmed and arXiv. Rivendell uses a new searching approch called "project-based search" designed by academics for academics.

Rivendell handles parall search for different projects by asking the user to explicitly divide she's searchs into groups (called projects). By explicitly marking search results as relevant and irrelevant, Rivendell learns the user's preferances and personalize the results over time for each project.

In addition, Rivendell has a "back door" for MNR related searchs - taking advantage of relevant marked search results from searches contins list of enetity words related to MNRs. In this way, Rivendell is gathering papers for Elrond's Library.

Nanosim Simulator

A mathematical model treating the in-vivo circulatory system as a flow graph and MNRs as final state machines (FSM) implemeneted as a stochastic graph-based simulator.

The Nanosim simulator is able to answer both prediction questions (aka. Forward queries) given the initial state of the system and the "what should happen to get" questions (aka. Backward queries).

Nanosim has been tested relative to in-vivo lab results, and found to provide very good initial results.
